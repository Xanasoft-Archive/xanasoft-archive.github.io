Hallo austin,

> Ginge das mit eurem Netloader ?

Mit ein paar kleinen Erweiterungen würde das durchaus gehen.

Willst du die FTP links in dem Format ftp://UserName:Password@server/dir1/dir ... n/clup.avi einfügen?
Oder soll der Client ein vorkonfigurierten FTP Path in bestimmten zeit Intervallen nach neuen Files durchsuchen und diese automatisch uploaden?

Automatisch Posten auf link crypt Seiten müsse ich adden, aber die web Engine welche der Neo nutzt ist so flexibel das das schnell zu machen ist.

Momentan ist das eigentlich so gedacht das der Neo die links verschlüsselt in einem dezentralen DHT Netzwerk speichert, und links in der Form
magnet:?xt=urn:neo:6-X6e247APYXcbjzTRoj8UcaNI4lQyFr8wr1WFFcmeo&dn=Readme.txt&xl=172
Der link ist für alle uploads eines binär gleichen files identisch, das File kann auch wie bei Torrents auch eigentlich ein Verbund mehrerer Files sein, also z.b. MeinUrlaub.avi und ReadMe.txt

Den link kann man dann anstatt eines link crypt Seiten links irgendwo Posten und wen ein downloader den link in seinen NeoLoader einfügt fragt der neo die aktuell neuesten Hoster links aus dem dezentralen DHT Netzwerk ab und lädt sie herunter.

Damit kann man links erstellen die für den downloader so lange gültig bleiben wie lange man immer wen irgend ein hoster link vom hoster gelöscht wird das file neu hoch lädt und den neo den neuen link ins DHT einfügen lässt.

Die Parts kann man auf 2 arten hoch-laden als normales split-archiv (optional mit passwort) oder in einem neo spezifischem format, welches die uploads individuell verschlüsselt, damit kann man sachen auf ein und den selben hoster reuploaden so das der hoster nicht merken kann das das file schon mal hochbeladen wurde.
Ein weiterer Vorteil des Neo Systems ist das nicht alle teile gleich groß sein müssen, z.b.
Du hast file X 1 GB groß das lädst du auf hoster A in form von 2 400MB und 1 200MB großen files, und auf hoster B als 5 200MB files,
Jetzt wen das split Archive wären musste sich der downloader für ein hoster entscheiden da die parts nicht kreuzt kompatibel wären, mit dem neo System sind die das aber,
er hat 1 400MB file von hoster A und 2 200MB von hoster B laden dazu das letzte 200MB part egal von wo und daraus sich das file zusammen stückeln (also das macht der neo ganz automatisch).

Aber wie gesagt eine auto post Funktion auf crypt seiten kann ich relativ schnell adden, welche crypt seiten müsten den unterstützt sein?
