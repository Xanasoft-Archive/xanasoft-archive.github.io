Hi, thanks for trying neo

1) This option will block neoloader on the network that you choose here. Basically using a VPN, you should lock neo on it, so if it will go offline or there's any problem neoloader will stop working instead of use the network without VPN (showing your real IP). I am not sure about that, because I never used a VPN. About the "Ethernet" option, I don't see it, probably is due to windows configuration or driver, I am not sure here either. And yeah, without a VPN there's no reason to lock neoloader on a certain adapter, leave it blank... for the torrent not downloading though try to lock it to Eth or LAN (restart after this change) and check if it will find seeds. If the problem persist, leave it blank it should be blank without a VPN.

2) It affect the ed2k/kad network only. You can find some infos here: http://www.emule-project.net/home/perl/ ... show_topic
The usage of a static hash, like eMule, gives the benefit of the credit system avaible in any eMule client but you can be traced even between an ip change since this hash always remain the same (if you don't make a "clean" install, may stay the same for years). Even if in past this tracing never happened, is theorically possible and since our policy is against the credit system (I'll not explain why this night), by default we use a random hash. It should change if you restart neoloader after 6h, in that case on startup neo will choose a different userhash iirc.
