Postby DavidXanatos » Sat Nov 09, 2013 7:12 pm

yes, I don't think its a bug.
I tested it and with a 100% random file the countes show the same value.

The mismatch occures because eMule counts the amount of packet bytes send on the network, while neo counts the amount of file bytes transmitted.

When the file data gets compressed you need less packet bytes to transmit a certain amount of file bytes.
So the better a file can be compressed the larger the missmatch in countes will be.

Should I change neo to count always packet bytes?
I belive counting file bytes is more correct, but I guess that depands on what one wants to know.

EDIT: I think I'll change it to count emule style so that there wont be a confusion about that.
