0.31a fix list will be uplaoded soon

- Unstable download speed, same as upload, tested with linux distros. Upload is really really bad regarding torrent.
-- Stability improved in 0.31a using using more mule like timins on socket ratecontroll.
- Ed2k download is not working
--Should be fixed in 0.31a
- NeoKad search is not working (verified by me, reported by: enOehTsIsihT, thanks)
-- NeoKad should connect faster in 0.31a
- NeoLoader hashes every file that found with MuleKad search!! You can see this in log, this have to be fixed asap.
-- fixed in 0.31a
- Hashes every time shared files on startup
-- fixed in 0.31a
In other words the duplicate status should not be propagated to the multi file torrent containing the duplicate.
- Video Player is not working, I wish you will remove it :\
-- In the new updater/feature manager there will be an option wather to install the player or not.
- Crash (with .dmp) adding a torrent file. [Magnet links work correctly that's why I don't mark red this one]
-- fixed in 0.31a
- Double log line on multiple torrent corrupted part. 1 for folder that is useless and 1 for file that is correct/useful one
-- that is intended this way, for a normal multi file torent it does not make much sense, but when there will be more files in kad and the sub files will get ed2k hashes associated befoure completion its important to have logging for booth sub files and parent multi files.
- When paused files it still download/upload datas. Stop work correctly
-- the download will nto stop in place, outstanding requests will be allowed to compelte also uploads will be finished.
- ed2k file sharing bug --> NeoLoader 0.31 Bug List (thx @enOehTsIsihT)
-- It actually is intended this way, the Share on Ed2k flag is set when the first time a file is seed by Neo, and it remains as it is set for ever unless the user changes it manually.
By deactivating default share on ed2k files that was granted ed2k shgaring will not be revoked this status.
