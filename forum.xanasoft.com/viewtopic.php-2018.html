This should work by removing the template that gives direct access to the browser bookmarks from the configuration of that box.

Code: Select all

Template=[nameofthebrowser_Bookmarks_DirectAccess
Within Sandman (the default user interface for Sandboxie Plus) you can do that in the box options under "App Templates", while looking for the category "Web Browser", where you can disable the direct bookmark access. There are similar settings for other browser resources.

I believe that the default is set the way it is, to move from sandboxed to unsandboxed without having any differences.

by bastik-1001
I don't see why it should fail.

Just wanting to point out that processes need to be restarted in order for changes to take effect, in case, someone wants to try to recreate that.

With your set-up, sandboxed Brave should be able to read the bookmarks of the non-sandboxed Brave, but unable to modify them for the non-sandboxed Brave.
CopyBlockDenyWrite=y return a deny write error for all programs that try to open a file that can't be migrated
You have set the above, so it should not be possible for the sandboxed Brave to copy the bookmark file into the sandbox. Normally, when a sandboxed program tries to write to a file Sandboxie creates a copy in the sandbox, so the program can write to the copy and then will also read it from there. I am not sure how that behaves when there is already a copy in the sandbox or if the program creates a new file on write errors. Anyhow, it still should the case that sandboxed Brave can read the bookmarks of non-sandboxed Brave, while the non-sandboxed Brave should not be able to read changes made by Brave inside the sandbox.

You will have to clear the sandbox (delete its contents) to get rid of what has been stored inside.

You can try adding the following to the section [DefaultBox], but it should not change the behavior I described.

Code: Select all

ReadFilePath=path to the folder where the bookmarks are stored
The following, added to [DefaultBox], will have the effect that sandboxed Brave can't read the bookmarks of the non-sandboxed one. It also prevents writing to the file outside the sandbox (which it should not do to begin with, as there is no OpenFilePath).

Code: Select all

ClosedFilePath=path to the folder where the bookmarks are stored
