  • Quote

Post by Pebbles » Thu Aug 04, 2022 6:13 pm

Problem: the VST plugin suite "Captain Plugins" does not work in the DAW FL Studio, when running in sandboxed mode. The plugins only display a black window. They work fine if installed and run on a system where the host is unsandboxed.
I use an application compartment sandbox. Firewall is open both inside and outside the sandbox.
Sandboxie-Plus 1.2.7, Windows 10.

To reproduce:
1. Download and install in a sandbox FL Studio trial from here: https://www.image-line.com/fl-studio-download/
2. Install to same sandbox the VST plugin "Captain Plugins" which can be downloaded here: https://mixedinkey.com/captain-plugins-download/
3. Inside FL Studio, search for VSTs on the system, flag the Captain plugins for use (Captain Chords, Captain Melody, etc). (This step is probably the trickiest, if someone is not at all familiar with DAWs or FL Studio.)
4. Add one of the plugins to the channel rack in FL Studio, for example "Captain Melody".

The plugin window now comes up, but is only black. All other plugins I've tested work, but the Captain Plugins are a bit more involved. Aside from the regular vst .dll or .vst3 file that make up most plugins, the Captain suite comes with a bunch of extra files and dependencies. They also need to go online in order to function properly, because they communicate with a server for part of their functionality.
