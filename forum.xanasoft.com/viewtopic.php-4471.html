A strange error: Non-sandboxed windows appear to be sandboxed.

  • Quote

Post by Sandboxer » Fri Jul 30, 2021 12:43 pm

I only use my five sandboxes to run browsers (Vivaldi, Firefox, Edge and IE). I usually force all browsers to open in DefaultBox, and I never start any other program sandboxed.

For the last couple of months, 0-5 times a day, the yellow border of DefaultBox appears around some other program's window. Today I recorded a few things when an Excel spreadsheet received a yellow border.

* First, I used Sandboxie's Windows Finder, which reported that:
The selected window is running as part of the program Vivaldi.exe in sandbox DefaultBox.
This confirmed the yellow sandbox DefaultBox, but astonishingly it claimed that it was Vivaldi that was running in the sandbox! Vivaldi is my default browser, and it was not running at all, although it had been earlier.
* I then emptied DefaultBox (including system files) with normal Windows delete.
* I then entered some new text into an Excel cell, and saved the file with Ctrl+S.
* After refreshing the directory tree, the sandbox was still empty, and Excel still had its yellow border.
* I then closed Excel, and reopened the same file. This time there was no yellow border, and Sandboxie's Windows finder reported that:
The selected window is not running as part of any sandboxed program.
Also, the change that I had made within the Excel file had been properly saved.

* Twenty minutes later I launched the XLSX file again --- it had a yellow border.
* I closed it and reopened it --- no yellow border.

I am not a techie and I don't understand what is happening, so I can only report the behaviour.
64-bit Sandboxie-Plus V0.8.9 running on a 64-bit desktop (Intel Core i7-6700 CPU @ 3.40GHz) with Windows 10.0.19043.1110.

This error is still a problem for me in V0.9.3. The only changes that I can detect are that it usually seems to be happening less, and that the Window Finder now confirms that the window is sandboxed according to the colour of the border.

Only my four browsers are forced into DefaultBox, and only The Bat! (my email client) is explicitly prevented from opening in a sandbox.

But this afternoon, Sandboxie really seems to have caught the dreaded Delta variant! Here is what happened on the same PC mentioned above.

* I launched ESET Internet Security by left-clicking its System Tray icon.
— Result: Launched in DefaultBox (yellow border, confirmed by Sandboxie Window Finder).
— I repeated this twice more, with the same result.
— I then launched ESET by left-clicking the system tray icon. It launched unsandboxed.
— It also launched unsandboxed using a button with the Start Menu LNK file.

* I then launched this Forum webpage in DefaultBox, and it launched correctly.

* I then launched portable Neo's SafeKeys using a button with its command.
— Result: It launched in DefaultBox, confirmed by border and Windows Finder.
— I repeated the launch twice more, and it came up unsandboxed each time
— But the fourth time it launched in LoginBox, confirmed by border and Finder.
— Launches 5–7 came up unsandboxed.

* I then launched RoboForm using a button with its Start Menu LNK. It launched unsandboxed.
— The next time it launched in LoginBox.
— The third time it launched in DefaultBox.
— The fourth time it launched unsandboxed.

At this stage, I quickly logged into RoboForm, then into the Forum, and posted this note.
