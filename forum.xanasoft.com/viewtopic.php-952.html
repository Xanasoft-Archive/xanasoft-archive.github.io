by DavidXanatos
The feature comparisons page already posted explains it mostly already.

The confusion about the expiration of Personal (formally Medium) certificates stems from how they work,
The certificate is valid for the current build and any build released less than 1 year from the date of issuance.
For those builds it will never expire with regard to unlocked functionality.

They will however expire in the sense as they won't be valid for a build released 1 year and 1 day from the date of issuance.

Also, there is another aspect recently added in 1.10.0 the troubleshooting wizard which comes with a library of troubleshooting scripts will only download new scripts from the server when the certificate is not yet expired. Likewise, the new updater option to download the latest Compatibility templates will only work with a non expired certificate.

I really should
1. mentioned the new benefits on the webpage (these functionalities were not present in older builds at all)
2. explain that it needs a non expired certificate to work

Also for completeness there is the Eternal (formally huge) certificate which never expires, but it's ridiculously expensive.

If the advanced encryption upgrade applies for 1 seat or for multiple depends on the certificate type being upgraded.
When you upgrade a business cert you need one advanced upgrade per PC,
When you upgrade a personal, subscription, or family pack you only need one upgrade key for all the PC's

I am under the assumption that if one purchases the "Personal" certificate and the advanced upgrade, that this person can apply the personal certificate to all his devices AND apply the upgrade to all of them.
This is correct.
Further, the word pack, on the advanced upgrade license, suggests that it can be used as a pack with the family pack. One buys a family pack license and ONE advanced upgrade license, and could be able to upgrade all family computers. You could clarify if that is the case.
This is the case.


The way the mechanism works is: when you enter a upgrade key the UI sends a cert request to the server providing the update key from the installed cert and the serial number. The server validates if the serial number exists, and checks if for this serial number a certificate was already created, if it was it just returns the created certificate as is, if it was not it retrieves the original cert by its update key and generates an upgraded certificate which is then stored and returned

So for the first family PC the key is entered the server will generate an advanced cert for any subsequent PC it will just return the already made one,
so you can just as well simply copy the upgraded cert from the first PC and enter it directly on any new family PC.
