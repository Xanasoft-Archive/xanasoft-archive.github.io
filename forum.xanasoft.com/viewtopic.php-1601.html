To speed things up you can use an iterative approach start with something like 5.40 if the bug is there than pick one in between 5.33.6 and 5.40 some ware in the middle, of the bug is not, than pick one in between 5.40 and 5.55.11 some ware in the middle, and so on... and then again in the midden and in the middle, this way you will need to test the minimum ob builds until you find the exact one that started the issue.
