As long as all boot relevant partitions are not encrypted, and you don't have secure boot enabled everything should be fine
So keep C:\ and the acompanying boot/uefi/recovery partitions unencrypted
the issue is that if windows denyes loading the DC driver even if nothing is encrypted it wil result in a BSOD on boot as the driver is registered as a low level filter driver. Windows is not smart enough to ignore that entry if it previusly declined to load the driver, in that case you can use a boot menu option to disable signature enforcement to boot once and uninstall dc
