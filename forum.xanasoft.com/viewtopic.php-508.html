ESET v17 new 'Browser Privacy & Security' feature permanently crash Firefox while sandboxed

  • Quote

Post by Bros » Thu Nov 23, 2023 10:57 am

Just a few days ago ESET Corporation launch all brand new major v17 of their Security Software products portfolio:

https://help.eset.com/changelogs/

And in this v17 (the first available version is actually v17.0.15.0 here...), ESET Corporation release all brand new Browser Privacy & Security feature:

https://help.eset.com/eis/17/en-US/idh_config_bps.html

https://addons.mozilla.org/en-US/firefo ... -security/

https://chrome.google.com/webstore/deta ... jkhicgfaam

https://microsoftedge.microsoft.com/add ... cnbomnaakc



The problem is that this brand new ESET v17.0.15.0 software products from now on permanently crash Mozilla Firefox web browser immediately while running sandboxed with Sandboxie (Classic v5.66.4) software!



The only solution for now and until Sandboxie software new version will properly support all brand new ESET v17.0.15.0 software products is to disable this whole brand new ESET Browser Privacy & Security feature.

To be able to do that please open your ESET v17 software product, then go to Setup -> Advanced setup.
On the left please click on Protections -> Browser protection, and temporary disable here the:

a) Browser Privacy & Security

b) Browser script scanner


After that disable Settings will temporarily be applied here, Mozilla Firefox web browser will run again just fine while sandboxed with Sandboxie software.



Additional helpful information here:

https://forum.eset.com/topic/38819-eis- ... sandboxie/

https://forum.eset.com/topic/38809-nod- ... rsion-119/
