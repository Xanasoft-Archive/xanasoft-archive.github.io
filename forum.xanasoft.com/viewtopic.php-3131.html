Saving files outside of sandbox directly

Posted: Sat May 06, 2023 9:19 am
by n4b7
I have an application running in a sandbox and want to store some files (via the application's save as dialogue) outside of the sandbox directly, without using Sandboxies file recovery utility.
I therefore specified a directory (with access mode: open) under the sandbox options > Resource Access > Files
However, the files are still not stored directly on the disk, I still have to go through the recovery dialogue.
What am I doing wrong?

Re: Saving files outside of sandbox directly

Posted: Mon May 08, 2023 4:56 pm
by bastik-1001
If the application is located outside the sandbox (or even installed outside the sandbox),

Code: Select all

OpenFilePath=your_path
should work.

However, if you created a box and downloaded an application (or even installed it inside the sandbox), then making the folder accessible via OpenFilePath does not work, for security reasons. Using

Code: Select all

OpenPipePath=your_path
allows programs located inside the sandbox to write to that path.

Edit: In the modern UI, the access type is called "open for all". And here is the documentation for OpenPipePath.
