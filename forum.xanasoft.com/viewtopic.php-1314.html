It is most likely a false positive. I am not sure, when there will be another final release. Probably only David can tell.

No detection does not always mean that a file is clean, but at least there is a better feeling.

There have been reports of hits on unsigned pre-releases. The build is properly signed, which in itself is not a valid reason to blindly trust some piece of software, but at least it could not have been modified after it was released. I expect the rate of hits to be higher if someone managed to place malware in this software bundle.

You could wait a few days and check again, if the warning is gone. As malware samples are shared among the vendors they should be able to confirm it and the hits would increase if there is something fishy going on.

You also could reach out to Ikarus, via e-mail fp[-a-t-]ikarus.at or false-positive[-a-t-]ikarus.at (The [-a-t-] need to be an at-sign [@] to see if they can confirm it to be a false positive.

by bastik-1001
Signature updates vary from product to product, but usually it does not take long. For consumer products, some release several signature updates per day, others once per day. I don't know at which rate they are updated for VirusTotal where it is often not configured to be as close as what end users will see. They can be configured to be more aggressive on VirusTotal, as a false positive is nearly harmless, compared to dozens or hundreds or even thousands of complaints if a product raises too many false alarms or even deletes legitimate files.

I checked, before typing the reply, which is far later than you asked to wait for the flag to be cleared, and now there are no more reports for the file in question. Edit: If they confirmed something to be a false positive, there would have been little reason to wait, in my opinion.

It's very nice that they reacted so promptly and that they were so kind to you.
