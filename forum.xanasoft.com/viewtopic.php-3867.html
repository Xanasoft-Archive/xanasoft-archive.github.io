David, I'm sorry to report two further problems.

I updated Sandboxie-Plus From V0.8.9 to V0.9.1 on 2021-08-03 (I dodged the beta V0.9.9a) on my 64-bit desktop running Windows 10.0.19043.1110. My four browsers are all forced to open in the default sandbox.

Before installing V0.9.1, everything was running smoothly, apart from the occasional recent 'strange error' of a boundary around non-sandboxed windows, as described in my post above at
viewforum.php?f=12.

FIRST PROBLEM: After intalling V0.9.1, the command
Sandboxie /dfp Vivaldi
does not launch Vivaldi at all. Instead I get an error message:
PID 3608: SBIE2335 Initialization failed for process vivaldi.exe [44 / 1605]
I can't find PID 3608 or SBIE2335, and I don't know what the words mean.

WORKAROUND: If I temporarily disable forcing, the command above works as it should.

SECOND PROBLEM: I can usually launch Vivaldi in a sandbox OK, using for example the command
Sandboxie /Box:TrueBox Vivaldi
except that sometimes when I launch Vivaldi in another sandbox, it launches instead in the DefaultBox.

WORKAROUND: Do something else, then try it again and it usually works OK.
