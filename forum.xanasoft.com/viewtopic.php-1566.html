Deleting the line from the ini file was unfortunately not the solution after all. I could then start Thunderbird again, but not save anything.... After a longer search I have now found the solution and would like to post it here. Maybe it is interesting for someone else:

The default path for Thunderbird is "%AppData%\Roaming\Thunderbird".

But I moved everything, just to "D:\Eigene Dateien\Office\..." I didn't mention that here because I didn't even remember it. When I set this up in Sandboxie-Classic a long time ago and allowed this path for direct access, it worked. It also worked with Sandboxie-Plus at first.

What I forgot about this is "%AppData%\Roaming\Thunderbird".
There I simply deleted the Thunderbird folder and replaced it with a Symbolic Link!

Sandboxie-Plus as of Verson 1.x apparently now handles Symbolic Links differently.
Additionally allowing direct access to "%AppData%\Roaming\Thunderbird" did not solve my problem though.
I also have to delete "D:\Eigene Dateien\Office\Thunderbird\AppData Roaming\Thunderbird" from the configuration.

Now everything works again even with the current versions!
