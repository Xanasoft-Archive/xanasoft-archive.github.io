The user and contributor offhub (on Github) concluded that the browser is set up to sync its data. Something I had thought not to be the case, since it is to be expected that when the profile is set up to sync data and said profile is used in the sandbox that the browser will sync itself.


To test this the following line needs to be added, so that the sandboxed browser will not be able to see the profile and any attempt to write to it the folder will be redirected into the sandbox. Please clear out the box, before trying.

Code: Select all

WriteFilePath=%LocalAppData%\BraveSoftware\Brave-Browser\User Data\Default\

I am echoing offhub, here, so that information can be found easily.

You create a separate profile for Sandboxie and use that profile under Sandboxie.

For regular shortcut:

Code: Select all

"C:\Program Files\BraveSoftware\Brave-Browser\Application\brave.exe" -profile-directory=Sandboxie
For sandboxed shortcut:

Code: Select all

"C:\Program Files\Sandboxie-Plus\SandMan.exe" /box:DefaultBox "C:\Program Files\BraveSoftware\Brave-Browser\Application\brave.exe" -profile-directory=Sandboxie
Or Run Menu entry for Sandboxie-Plus: (Sandbox Options > General Options > Run Menu) which will add this to the Sandboxie.ini

Code: Select all

RunCommand=Brave|"C:\Program Files\BraveSoftware\Brave-Browser\Application\brave.exe" -profile-directory=Sandboxie
