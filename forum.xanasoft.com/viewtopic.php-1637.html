Sandboxie Classic v5.64.8 and with Adobe Acrobat Pro v2023.003.20215 or more newer is crashing

Posted: Sun Jul 16, 2023 7:43 pm
by Bros
The Adobe Acrobat Pro (32-bit Continuous release) and with last two available updates for it (the v2023.003.20215, and the v2023.003.20244), when run with Sandboxie Classic v5.64.8 is crashing within 1 minute after the Adobe Acrobat Pro has been sandboxed, and with this error report:

Code: Select all

SBIE2224 Sandboxed program has crashed: Acrobat.exe [DefaultBox]
SBIE2112 Object is not accessible: Unnamed object, call ConnectPort (C0000022) access=001F0000 initialized=1
SBIE2112 Object is not accessible: Unnamed object, call ConnectPort (C0000022) access=001F0000 initialized=1



To be able to successfully reproduce the above bug error, I suggest to run sandboxed Mozilla Firefox Web browser (64-bit), and with the Adobe Acrobat extension installed here:

Code: Select all

C:\Program Files (x86)\Adobe\Acrobat DC\Acrobat\Browser\WCFirefoxExtn\WebExtn\signed_extn\adobe_acrobat-1.0-windows.xpi
This is the normal way how the Adobe Acrobat Pro software should function by default, because on this way numerous web pages across Internet
can be directly converted to the PDF file format, but from within the web browser itself.

So, when this PDF conversion process is finish here, that will automatically open sandboxed Adobe Acrobat Pro software also,
and with this just converted PDF file opened in it...

The problem now is, that within a minute and from last two above mentioned Adobe Acrobat Pro Updates, this sandboxed Adobe Acrobat Pro software is automatically crashing, resulting with the above Sandboxie error report.


However, if we just for e.g., simply try and press the right mouse click on the Adobe Acrobat icon on our Desktop and run it as Sandboxed,
this will not produce the above mentioned crashing at all, so someone can conclude that everything is working as supposed to be here, but unfortunately is not...




Thank you for help and advice, for fixing this bug also, and best regards!

You welcome bastik-1001, and thank you very much also, for pointing this all out, that all this can be quite possible connected problems here...


Well, there are actually 2 ways how to create with Adobe Acrobat Pro software the Adobe PDF files from the Web pages (within a Web browser...):

a) The first and most desirable way is, as I already explain on the begging of my first post here, with Adobe Acrobat extension, and that process will give as a fully "editable and interactive PDF/dynamic PDF" file at the end

b) When we open our Web browser, then go via File -> Print -> As the PDF file, and that will give us only a "static PDF" file at the end of the process here, and usually we less want that kind of the PDF file to get after all here - cause it is static and flat..., it is actually just one big picture converted to a PDF file format and that's all...


So, for this experiment, I also try today to get this of a static PDF file (the b) option mentioned here), but from sandboxed Mozilla Firefox Web browser...

At first 3-4 PDF files creation here on that particular way, apparently it will function OK, but after more and more PDF files here are being created like that (from different Mozilla Firefox Web browser tabs, different web pages...), during of this PDF file process creation, it will produce after all numerous Sandboxie errors which he will report..., but luckily and somehow, disregard all of this errors reporting, it will create simply all the static PDF files here and after all:

Code: Select all

SBIE2203 Failed to communicate with Sandboxie Service:  *GUIPROXY_00000001; MsgId: 12 - acrotray.exe [C0000024]
SBIE2203 Failed to communicate with Sandboxie Service:  *GUIPROXY_00000001; MsgId: 12 - acrotray.exe [C0000024]
SBIE2203 Failed to communicate with Sandboxie Service:  *GUIPROXY_00000001; MsgId: 12 - acrotray.exe [C0000024]

However, it will again automatically just after every successful creation of that static PDF file(s) here, it will open it in sandboxed Adobe Acrobat Pro software at the end, and again this sandboxed Adobe Acrobat Pro software will then crash within a minute, like it is already described in my very first post here...


... Except (now, this is very interesting!), if we first run sandboxed Mozilla Firefox Web browser as usual, then go manually to an Adobe Acrobat (Pro) icon on our Desktop, and right click on that icon, and then run it Sandboxed like that.

Now, if we done everything and exactly like that just being said, the Adobe Acrobat Pro software will not crash sandboxed anymore like before - like when we open first the sandboxed Mozilla Firefox web browser, and then start generating over Adobe Acrobat Pro software - but within Mozilla Firefox Web browser, the PDF file..., and as soon this PDF file is created, it will open it in sandboxed Adobe Acrobat Pro software, and then this very same and sandboxed Adobe Acrobat Pro software will crash within a minute in his sandboxed mode...

Well, the problem is that the normal behavior, is the last one mentioned procedure here and for generating the PDF files with Adobe Acrobat Pro software in the first place (so, when we open first the sandboxed Mozilla Firefox web browser, and then start generating over Adobe Acrobat Pro software - but within Mozilla Firefox Web browser, the PDF file(s)...), and this procedure function just perfectly sandboxed, until last two Adobe Acrobat Pro updates...



So, I believe that all of this Adobe Acrobat errors now, are actually just related here...
