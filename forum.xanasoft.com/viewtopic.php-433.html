I've recently setup Diskcryptor on a new Windows 11 PC. I need help now to fix boot problems.

Several Windows updates were installed including a BIOS firmware update. A restart was required, and all was fine.

1) The update required a reboot, which worked fine, required password, and then started showing a percentage progress.
2) It auto-restarted itself, at which point the machine went into Windows automatic recovery. This failed of course.
3) I manually restarted and had to press F12 and select the "DiskCrypto (DCO) loader" boot option. It prompted for a password, and then showed a new percentage progress indicator.
4) The same thing happened as in Step 2, rebooting again, failed recovery, and I needed to manually select the "DiskCrypto (DCO) loader" boot option.

Every attempt to boot the PC is causing this behavior now. It used to work fine before the update.
The default UEFI Boot Device shows "Windows Boot Manager".

How can I troubleshoot the current DiskCryptor configuration to fix this?
