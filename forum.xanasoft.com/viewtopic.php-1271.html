Did that happen across multiple versions? I imagine there being an issue with the installer in some cases, but it shouldn't be reoccurring, unless there is a flaw in the updater's logic, in which case I'd expect it to be triggered for multiple people.

You might want to back up your sandboxie.ini (usually located in C:\Windows; holding the configuration of Sandboxie and the sandboxes) the sandboxes themselves (if any, usually at C:\Sandbox\), maybe the sandboxie-plus.ini (containing UI settings, usually located at %AppData%\Local\Sandboxie-Plus\), your certificate.dat (if any) located in the installation folder of Sandboxie-Plus, then uninstall Sandboxie-Plus, just to do a clean installation.

Please note that David changed the update procedure from within the user-interface for non-active, non-valid, certificates, so that the update from stable release to stable release is no longer automated. It requires a certificate to do an automated installation. Without a valid and active certificate, users are guided to the download page. Pre-releases are not covered. This change got made with Sandboxie-Plus 1.12.4.
