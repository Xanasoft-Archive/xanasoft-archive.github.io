Posted: Sun May 07, 2023 6:26 pm

During fast file retrieval from a browser opened in the virtual area, if the file to be recovered has the same name as a file already present in the destination path, the file overwrite request window is not placed in front of the recovery window, which can hide it until it is blocked, if the window is too large, such as in the case of a high DPI. In the video the recovery window partially covers the confirm/reject window, but sometimes it happened to me that it completely covered it, requiring me to use the Alt+tab combination to be able to turn it off, and regain control.
