Hello, the releases on the main website are purposefully lagging behind to catch if something is wrong with them, but it's unclear to how far it is supposed to lag behind.

As of now, the forum announcements only list final releases, which I think is a good thing, as not everyone is interested in pre-releases. Curiously enough, 1.9.7 has been followed by another stable release namely 1.9.8, which has not been announced here.

The releases are hosted on Github, anyway. The announcement for 1.9.7 links to
(https://github.com/sandboxie-plus/Sandboxie/releases/tag/v1.9.7),
while the latest stable is 1.9.8 (at the time of writing, released 21st June 2023)
(https://github.com/sandboxie-plus/Sandboxie/releases/tag/v1.9.8)

Following the latest link shows a page, where users can select the version they need for their system. In most instances, that will be Sandboxie-Plus-x64-v1.9.8.exe, since it is expected that they have a 64bit system. (x86 is for 32bit systems, while ARM64 is a different architecture)

However, there should be no need to visit a website to update to the latest version, unless you want to install it all over again. Within the user-interface, you can select "Help" and then click on "Check for Updates". Starting with 1.9.7, you can add an icon to the toolbar for that function. In the main options of Sandboxie-Plus' user-interface, you can enable to search for updates regularly. It is even possible to select which versions you prefer, like stable or preview. In your case, that should be the stable versions. One is also able to configure the workflow of updates, like only notifying or downloading an update for you. Upcoming releases, including 1.10.x will provide a selection of intervals, for how often to check for updates.
